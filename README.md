# CSN6214 OS Assignment - Producer-Consumer Transfer & Parallel Analytics

Group project for CSN6214 T2610 (Operating Systems). Transfers a large file
of integers over sockets using forked processes, then sorts it and finds
min/max using threads.

## Content

**Part 1 - server/client:** server.c reads a binary file of int32 values and
splits it into N chunks, served over TCP (port 9090). client.c forks N child
processes, each grabbing one chunk and writing it into a shared mmap'd
buffer, then puts the file back together into reassembled.dat.

**Part 2 - operations:** gets launched automatically by client.c once
reassembly is done. Spawns T threads to find the global min/max, then sorts
everything with a parallel merge sort (threads sort their own chunk
first, then get merged back together in pairs).

## Building

```bash
make
```

Builds server.c, client.c, and operations.c. make clean wipes the binaries
plus anything generated by a test run (original.dat, reassembled.dat, logs,
etc).

## Running it

Start the server.c first, pointing it at your input file:

```bash
./server original.dat &
```

Then run the client, telling it how many chunks (-p) and how many threads
for Part 2 (-t):

```bash
./client -p 4 -t 8
```

The client then handles calling operations for you once the file is
reassembled.

## Testing

```bash
./run_test.sh
```

Generates a 1GB test file, runs the whole pipeline end to end, then
checks the output files against what they should actually be (md5sum for
the reassembly, and independently recomputed min/max/sort for Part 2). Each
check prints PASS or FAIL so the user can see what broke if something did.
Needs python3 for the ground-truth checks.

## Output files

- reassembled.dat - should be byte-identical to the input file
- result_min.txt / result_max.txt - MIN=<value> / MAX=<value>
- result_sorted.dat - the sorted version, ascending
- execution_log.txt - logs which sync primitives or thread counts were used
